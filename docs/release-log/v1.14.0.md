# TokenPak v1.14.0 Release Log

## Scope

This release packages the independently accepted Codex shared-history
contention bridge: typed fail-closed diagnostics, an interactive temporary
session choice after verified contention, and additive accounting evidence that
states when prior shared history is not attached. It preserves the existing
default refusal and all non-interactive behavior. It also removes the accidental
Anthropic API-key/model prerequisite from the first-value journey by adding an
already-authenticated Codex OAuth Responses path. Version v1.13.1 was never
released. The final repair also restores the supported `/health` contract,
preserves the deprecated route-mixin compatibility payload for one window, and
separates cold listener-admission evidence from warmed sustained latency.

Linked work:

- Staging PR #601 — verified Codex contention recovery.
- The v1.14.0 release-bump staging pull request.
- The final health-compatibility and benchmark-contract repair.

## Public-surface compatibility ruling

- **Keep:** `PreflightStatus`, `PreflightEvidence`, `FallbackDecision`,
  `PreflightEvaluation`, and `TemporarySessionChoice` are additive beta API
  symbols and are included in the generated public-API snapshot.
- **Keep:** the new `tokenpak_setup` receipt members are optional additive
  evidence. Existing members and meanings are unchanged; consumers that read
  selected fields continue to work.
- **Keep:** the canonical basic `/health` schema remains the supported wire
  contract and is computed per request. `?deep=true` is additive. The
  deprecated route-mixin payload and one-second cache remain available for one
  compatibility window.
- **Breaking changes:** none.
- **Deprecations:** none.
- **Migration:** none.
