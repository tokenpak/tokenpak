## What's In v1.1

### 🔐 Security: Fixing query_preview

We found this ourselves in our pre-launch security audit, and it's the right thing to fix immediately.

The routing ledger currently stores the first 200 characters of each prompt in SQLite for debugging purposes. That's prompt content — and it shouldn't be there.

**Fix:** Replace `query_preview` with a short hash (`sha256[:16]`). Useful for deduplication tracking, zero prompt exposure.

This ships in v1.1. No action needed from you after upgrading.

### ⚡ Performance: 20% Faster Compression

We're profiling the compression pipeline and targeting a 20% speed improvement on the hot path. You'll notice it most on long inputs (10K+ tokens).

Benchmark results will be published when v1.1 releases.

### 🛠️ API: Developer Ergonomics

Building the community examples made us realize the current API is too verbose for quick use. We're adding:

**`tokenpak.compress()` shorthand:**
```python
# Before (v1.0)
engine = HeuristicEngine()
hints = CompactionHints(target_tokens=2000, mode="aggressive")
result = engine.compact(text, hints)

# After (v1.1)
import tokenpak
result = tokenpak.compress(text, target_tokens=2000)
```

**`TokenPakSession` context manager:**
```python
# Multi-turn conversations, simplified
with tokenpak.Session(model="gpt-4o", target_tokens=4000) as session:
    session.add("user", "Summarize this document")
    session.add("document", long_doc)
    response = session.send()
```

**Async support:**
```python
result = await tokenpak.compress_async(text, target_tokens=2000)
```
