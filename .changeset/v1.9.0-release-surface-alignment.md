Release-gate: public-API snapshot alignment for v1.9.0.

Regenerates the public-API snapshot to match the **released package surface**.
Two distinct changes, neither a hidden public-API expansion or silent removal:

**1. v1.9.0 onboarding surface now captured (+68 symbols).** The new
onboarding-lifecycle commands genuinely expose these public module-level
symbols, now recorded in the snapshot:

- `tokenpak.cli.commands.permissions.*` (25) — permission-tier verb surface.
- `tokenpak.cli.commands.config_env.*` (12) — `config doctor` / `config env`.
- `tokenpak.config.load_order.*` (9) — documented config load-order.
- `tokenpak.cli.commands.menu*` (18) — hardened menu renderer + lifecycle.
- `tokenpak.cli.commands.doctor.*` (3) — doctor lifecycle helpers.
- `tokenpak.cli.commands.uninstall.*` (1) — uninstall command.

**2. Release-surface alignment for dispatch + third-party re-exports.**

- The snapshot generator (`gen_api_snapshot.py`) now excludes the **dispatch**
  subsystem (`tokenpak.orchestration.dispatch*` + `tokenpak.cli.commands.dispatch_cmd`),
  mirroring the wheel's `pyproject` `packages.find` exclude. Dispatch stays
  preview / main-only and is **not** recorded as public released API — so the
  290 dispatch source symbols are deliberately NOT added here.
- Two incidental third-party re-exports are dropped (deterministically filtered
  by the generator's package-ownership check): `vector_local::faiss` (optional
  `faiss` dependency) and `websocket_proxy::WebSocketServerProtocol`
  (`websockets`-library symbol). Neither is TokenPak-owned public API.

Also fixes a latent `F821` (undefined `name` → `package_name`) in the snapshot
generator's top-level import-error path.
