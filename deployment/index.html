    TokenPak Deployment Guide¶

    This guide covers deploying TokenPak in various configurations, including single-machine, multi-agent fleet, and Docker setups.

    For quick local setup, see QUICKSTART.md.

    Network Topology¶

    Standard Fleet Setup¶

    TokenPak is designed to run as a proxy sidecar on each agent machine in a fleet.

    ┌─────────────────────────────────────────────────────────────────┐
│                        Tailscale VPN                            │
│                                                                 │
│  ┌───────────────┐    ┌───────────────┐    ┌───────────────┐   │
│  │    SueBot     │    │    TrixBot    │    │    CaliBOT    │   │
│  │  (Governor)   │    │  (Executor)   │    │  (Processor)  │   │
│  │               │    │               │    │               │   │
│  │ sue@suewu     │    │ trix@trixbot  │    │ cali@calibot  │   │
│  │ :8766 proxy   │    │ :8766 proxy   │    │ :8766 proxy   │   │
│  │ :8767 ws      │    │ :8767 ws      │    │ :8767 ws      │   │
│  └───────┬───────┘    └───────┬───────┘    └───────┬───────┘   │
│          │                    │                    │            │
│          └────────────────────┴────────────────────┘           │
│                        Tailscale MeshVPN                        │
└─────────────────────────────────────────────────────────────────┘
                                │
                    ┌───────────┴───────────┐
                    │   Upstream Providers   │
                    │  Anthropic · OpenAI    │
                    │  Google · Ollama       │
                    └───────────────────────┘

    Agent Addresses¶

    Agent Hostname LAN IP Tailscale IP Proxy Port
    Sue (Governor) suewu 192.168.168.50 100.x.x.x 8766
    Trix (Executor) trixbot 192.168.168.103 100.x.x.x 8766
    Cali (Processor) calibot 192.168.168.100 100.83.165.113 8766

    Always prefer Tailscale MagicDNS (suewu, trixbot, calibot) over raw IPs. IPs can change; MagicDNS resolves via the VPN mesh.

    Configuration Files¶

    TokenPak config is spread across several layers. Here's where everything lives:

    File Locations¶

    Path What it is Who controls it
    ~/.openclaw/.env API keys and token env vars Push via push-secrets.sh from Sue
    ~/tokenpak/proxy.py Production proxy binary Never edit directly — deploy via script
    ~/tokenpak-dev/proxy.py Staging proxy Edit here, test, then promote
    ~/vault/01_PROJECTS/tokenpak/proxy.py Canonical source All edits go here first
    ~/vault/01_PROJECTS/tokenpak/configs/ Preset YAML/JSON configs Reference configs for common setups

    Environment Variable Precedence¶

    Environment variables are loaded in this order (highest wins):

    1. Process environment (systemd unit or shell export)
    2. .env in working directory
    3. ~/.openclaw/.env ← canonical for fleet deployments
    4. Config env block in openclaw.json
    5. Optional shell env import

    See configuration.md for the full environment variable reference.

    Deployment Workflows¶

    Single Machine (Local)¶

    cd ~/tokenpak
export ANTHROPIC_API_KEY=sk-ant-...
export TOKENPAK_PORT=8766
python3 proxy.py

    Test it: 

    curl http://localhost:8766/health

    Fleet Deployment (Multi-Agent)¶

    All three agents run independent proxy instances. The canonical source lives on Sue, and changes are deployed fleet-wide via script.

    Dev → Prod workflow:

    # 1. Edit staging on your development machine
vim ~/tokenpak-dev/proxy.py

# 2. Copy to vault (canonical source)
cp ~/tokenpak-dev/proxy.py ~/vault/01_PROJECTS/tokenpak/proxy.py

# 3. Commit to vault
bash ~/vault/06_RUNTIME/scripts/git-safe-push.sh "proxy update: description"

# 4. Deploy to all agents
bash ~/vault/06_RUNTIME/scripts/sync-tokenpak-proxy.sh --restart

    Deploy options for sync-tokenpak-proxy.sh:

    Flag Effect
    --check Drift detection only (no deploy)
    --restart Deploy + restart all proxies
    --dev Deploy to staging (~/tokenpak-dev/) only
    (no flag) Deploy to production only

    Restart a Single Agent Proxy¶

    # Via SSH
ssh trix@trixbot "systemctl --user restart tokenpak"

# Check status
ssh trix@trixbot "systemctl --user status tokenpak"

# View logs
ssh trix@trixbot "journalctl --user -u tokenpak --no-pager -n 30"

    Docker Deployment¶

    cd ~/vault/01_PROJECTS/tokenpak
docker compose up -d

    The proxy will be available at http://localhost:8766.

    See DOCKER.md for full Docker configuration options.

    SSH Connectivity¶

    Each agent must be reachable from the others via Tailscale. Verify with:

    # From CaliBOT → TrixBot
ssh -o ConnectTimeout=8 trix@trixbot "echo SSH_OK"

# From CaliBOT → SueBot
ssh -o ConnectTimeout=8 sue@suewu "echo SSH_OK"

    Troubleshooting SSH: - Run tailscale status to verify Tailscale is connected - Check ~/.ssh/config for correct host aliases - Ensure SSH keys are deployed: ssh-copy-id trix@trixbot - If Tailscale is down, fall back to LAN IPs (see table above)

    Health Checks¶

    # Proxy health
curl http://localhost:8766/health

# Proxy stats
curl http://localhost:8766/stats

# WebSocket telemetry (if enabled)
curl http://localhost:8767/

    Secrets Management¶

    All API keys are centrally managed on SueBot. Never edit ~/.openclaw/.env directly on Trix or Cali — it gets overwritten on next push.

    # Push secrets to all agents (run from SueBot)
bash ~/vault/06_RUNTIME/scripts/push-secrets.sh

# Push to single agent
bash ~/vault/06_RUNTIME/scripts/push-secrets.sh cali

# Push + restart gateways
bash ~/vault/06_RUNTIME/scripts/push-secrets.sh --restart

    See configuration.md for the full list of managed keys.

    Troubleshooting¶

    Issue Check
    Proxy not starting journalctl --user -u tokenpak -n 20
    Auth errors grep "^[A-Z]" ~/.openclaw/.env \| cut -d= -f1
    SSH timeout tailscale status + ping trixbot
    High latency /stats endpoint, check upstream timeout settings
    Test suite cd ~/vault/01_PROJECTS/tokenpak && python3 -m pytest tests/ -q --co

    For more, see TROUBLESHOOTING.md.
